Most pervasive skills in computer programming:

Computer Science

Computer science is about solving problems in a systematic step by step approach:
  1. Break down the problem in pieces.
  2. Precisely and mechanically describe a sequence of steps.
  3. These steps can be executed by a computer
Computers, Programs and their Language

A program is a set of specific orders that tell a computer what to do. Many programing languages like e.g Python need interpreters in order to work. These interpreters are programs that translate code into direct orders to a computer. Each programing language has its own grammar, that means a specific set of rules. It is very important for a programing language to be short and precise without any language ambiguity. The notation that is used to create this kind of concise language is called Backus-Naur Form.

Python Expressions

An expression is something that has a value. Expressions can interact using operators. An expression can be replaced by other expressions doing derivations. This video does a good job of explaining the Python grammar for "expressions".

Lesson 2: Variables and Strings

What is a variable in Python

We use variables in order to give names to values. Possible values are numbers or strings. There is a convention not to write variables with capital letters.

Strings in Python

A string is a sequence of characters surrounded by quotes. You can use double and single quotes → This makes indentation possible.
Using the “+” operator with strings means concatenation. You can index strings to select sub sequences from strings.
You can use the method find to find the position of substrings.

We can even change the value of a variable by re-assigning it to a different value later.

What's the difference between what the equals sign means in 2 + 3 = 5 and my_variable = 5?

In python “=” doesn't mean equal, it means assignment you should think of it as “←” rather than its meaning in algebra.

Lesson 3: Input> Funktion> Output

What is a function?

A function (also called ”procedures”) maps input to output. For example a function called square takes a number as an input and produces its square as an output.

What is the difference between making and using a function?

In Python code functions are made using the keyword def after that we put the parameters (a,b,c…) in parenthesis.
Then we write down the instructions how to process the input and the return assignment.
A function is used by writing its name followed by the values in parenthesis.

How do functions help programmers avoid repetition?

Functions process input and produce new input. You define them once and can use them over and over again.

What happens if a function doesn't have a return statement?

The output will be "None"

Lesson 4: Control Flow & Loops: If and While

Decision Making: If Statements

When a determined test expression is true the following indented block of code is executed. We can determine an else expression in case the test expression is false. If not the interpreter automatically jumps to the next statement.

While - Loops

A while- loop is a construct to do things over and over again. As long as a text expression is true the block keeps executing. When it turns false it jumps to the next statement.

Lesson 5: Structured Data: Lists & For-Loops

Structured Data

We can summon data in list elements. Lists can contain anything we want (e.g. strings, numbers, lists). We can reference to that data as a group or address only part of it by indexing.

Mutability

Lists unlike strings are mutable objects. That means we can change the value of a list after we created it.

The difference between append and +

append adds another element to an existing list. If that element is also a list it adds that list as one single element and not as the sum of its elements

+ does the opposite: similar as with strings it does concatenation adding element by element.

The Power of Abstraction

Servers

In simple terms, servers store information and are responsible for responding reliably to HTTP requests, be those get or post requests. Servers generally respond in two manners, static and dynamic responses. Static responses are, as the name implies, static files like images or prerendered HTML files that are simply returned to the browser. Dynamic responses however, build a response in real time based on the request through the use of web applications.

The Importance of Validating Input

Input validation prevents our site from being hacked. Servers, if not properly protected from spammers and hackers, can receive malicious data input from unverified Post requests, and validation allows the server to deny harmful data if certain conditions are not met. Validation is import for all websites that employ some sort of dynamic content.
To prevent hackers from “breaking” forms and input boxes coders use HTML escaping. It replaces certain characters with other characters to prohibit malicious users from entering server-altering code.
Another reason for the importance of validation is user expirience. It makes it easier for the user to enter meaningful data.

HTML Templates

A template library is a library to build complicated strings (most of the time html).We can create a template which is then inherited by all the pages that share the template's characteristics (e.g. titles, footers, navigation-menus). This way we avoid repetition and save time not rewriting the same code. Templates also allow us to keep different types of code separate. This increases readability and makes the code easier to modify.

What I learned in Course 5

Intro to HTML and CSS

Frameworks
A CSS framework consists of pre-prepared CSS code with pre- defined classes, that makes it easier to do standards-compliant web design.
Frameworks like Bootstrap contain a grid, which is extremely useful for positioning elements on a web site making use of negative space. Other useful features are an image-responsive class, which allows images to adapt to different screen sizes. You can also include basic JavaScript features using so called Modals.
For more customized use we might consider the possibility of coding our own framework.
Flexbox
CSS Flex Box Layout is the latest layout model coming to CSS3. It allows elements within a container to be arranged. It is one of many useful tools for Responsive Design.

Responsive Web Design fundamentals

4 Common responsive design patterns:
Column Drop is probably the easiest. At its narrowest viewport elements are simply stacked one over another in a single column that occupies the entire viewport. With increasing viewport size the elements start being positioned next to each other in the same row creating new columns. Once the columns are all positioned in one horizontal row and hit the max-width, margin is added to both sides. We code this using a flex-container and flex-wrap. Then we set up media-queries.

Mostly Fluid is very similar to Column Drop but more grid-like with more columns fitting in different ways.

Layout Shifter is probably the most responsive pattern. We use multiple containers and the order-attribute for more complex positioning.

Off Canvas: Instead of stacking elements vertically it places not frequently used elements off screen. We use this .e.g. to hide the navigation menu on smaller screens. Typically the user has to touch the hamburger-icon to open the hidden content.

Articles from alistapart.com
I wanted to get deeper into the design aspect of codingand looked for articles about responsive design on alistapart.com.I first found an article by Ethan Marcotte from 2010 where he predicts the great future importance of Responsive Design. It was very interesting to hear him talk about Fluid grids, flexible images, and media queries as radical new technical ideas for responsive web design. He makes an interesting comparison between web design and architecture. He emphasizes the unpredictable character of the web that makes it difficult to find long lasting solutions, whereas in architecture traditionally creative decisions are made with a more static approach creating spaces that may last for centuries. Marcotte suggests using fluid grids as a more future proof solution. His approach is layout agnostic and adapts to any browser’s window width. It is a beautiful solution. Later it is suggested in the comments to add css transitions into the mix. I am keen on trying out all that stuff!!!
After that I read a much more recent article on the same topic by Paul Robert Lloyd from last August.Five years later responsiveness is an established design principle. A new way of thinking to guarantee the best user experience. Making content available to a large variety of users becomes the most important guideline. This becomes pretty clear in W3C HTML Design Principles and I would like to quote from there :

"In case of conflict, consider users over authors over implementors over specifiers over theoretical purity. In other words costs or difficulties to the user should be given more weight than costs to authors; which in turn should be given more weight than costs to implementors; which should be given more weight than costs to authors of the spec itself, which should be given more weight than those proposing changes for theoretical reasons alone. Of course, it is preferred to make things better for multiple constituencies at once".

Lloyd underlines the importance of a framework in order to assure quality and appropriateness of web development. He doesn´t refer to a framework in the sense of software libraries or other abstractions concerned with executed code. He opts for a framework in form of design principles as defined by Google or GOV.UK. These guidelines are built on user empathy to assure a long term success of universal designs for erveryone regardless of age, ability, or status.
Another important point is to keep code simple and “share the toolbox” to favour a more collaborative interdisciplinary approach.
